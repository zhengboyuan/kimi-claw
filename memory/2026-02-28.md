# 2026-02-28 技术笔记

## 问题：批量数据回补内存溢出

### 现象
- 跑30天数据回补时，进程在第三天崩溃
- 日志显示 SIGKILL，内存占用过高

### 原因分析
1. `daily_v5.py` 内部使用 `ThreadPoolExecutor(max_workers=4)` 并发16台设备
2. 每台设备135个指标数据同时驻留内存
3. 数据不释放，内存累积

### 解决方案

#### 1. 添加显式内存释放
```python
import gc

# 每天处理完后
gc.collect()

# 每5天额外清理
if i % 5 == 0:
    gc.collect()
```

#### 2. 参考之前方案（run_v3_batch.py）
- 单设备顺序处理（非并发）
- 分批处理（每30天一批）
- 及时释放大数据对象

### 关键代码位置
- `workflows/batch_backfill.py`
- 修改点：循环内添加 `gc.collect()`

### 验证结果
- 30天数据全部回补成功
- 成功率：100%
- 内存稳定，未崩溃

### 经验
- Python 循环处理大数据时，必须显式 `gc.collect()`
- ThreadPoolExecutor 并发会累积内存，需谨慎使用
- 长时间运行任务，定期释放内存是必需的
