## V5.1 内存溢出修复

### 问题
批量运行 10 月数据时出现内存溢出，进程被 SIGKILL 终止。

### 根本原因
- `gc.collect()` 只在 `run_daily_v5()` 函数最后执行
- 批量循环调用时，每次调用之间的内存没有释放
- ThreadPoolExecutor 并发 16 台设备，数据累积

### 修复方案
在 `run()` 方法内部添加 `gc.collect()`：
```python
def run(self, date_str: str) -> Dict:
    # ... 所有处理逻辑 ...
    
    # 释放内存，防止批量处理时内存累积
    gc.collect()
    
    return report_data
```

### 修复提交
- hash: `522995f`
- message: "fix(v5.1): add gc.collect() in run() method to prevent memory accumulation"

### 验证结果
- 批量测试 5 天数据全部成功
- 无内存溢出，进程正常完成
